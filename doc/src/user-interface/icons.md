# Icons

For new user interface elements, you should **always prefer** using [Material Symbols](https://fonts.google.com/icons) over Material Design Icons.  

Material Design Icons are no longer maintained and will not receive new icons, while Material Symbols are actively maintained and will continue to receive new icons. Material Symbols also have a more modern design and automatically adjust their look based on the size to ensure they always are clear and legible.

To start using Material Symbols, use the `<Symbol>` component.

_Exception: When your element is part of a larger component that already uses Material Design Icons, you should use the same icon set for consistency. For example, the icons for Settings items currently come from MDI, so you can continue using MDI icons for new Settings items until we migrate all Settings icons to Material Symbols._
